# Пожары: дневная ML-модель по районам

`fire_ml.py` — полный цикл (запуск в venv):

```bash
.venv/bin/python ml/i9-fire-risk/fire_ml.py fetch   # архив FIRMS апр–окт 2021–2025 (кэш+resume) + погода районов
.venv/bin/python ml/i9-fire-risk/fire_ml.py train   # LightGBM: train 2021–2024, test 2025; метрики + модель
.venv/bin/python ml/i9-fire-risk/fire_ml.py today   # прогноз на сегодня → fire-ml-today.json (режим «ML-прогноз»)
```

## Постановка

Датасет «район × день» (174 района × апрель–октябрь × 5 лет = 186 180 строк).
Таргет: был ли в районе в этот день очаг VIIRS. Признаки: t_max, ветер_max, осадки,
серия сухих дней, сезонность (sin/cos дня года), исторический приор района
(лог-число очагов, считается только по train-годам — без утечки).

## Результаты (честный временной сплит: тест — весь 2025)

| Метрика | LightGBM | Композит (продуктовый baseline) |
| --- | --- | --- |
| ROC-AUC | **0.832** | 0.686 |
| Average Precision | **0.647** | 0.386 |
| Capture@top-20% | **55.2%** | 36.2% |

Интерпретация для слайда: направляя патрули в топ-20% приоритетных район-дней,
модель накрывает 55% реальных пожарных дней — в 1.5 раза больше, чем эвристика.

## Прод

Модель сохраняется в `data/processed/fire_lgbm.txt`; `today` скачивает прогноз погоды
Open-Meteo, скорит районы и пишет `frontend/public/data/fire-ml-today.json` —
на карте появляется режим «ML-прогноз» (переключатель в шапке страницы).
Регулярное обновление (cron раз в сутки) — задача основной программы.
