# Зимняя ML-модель (район × день)

LightGBM предсказывает вероятность опасного зимнего дня для района (target —
предупреждение Казгидромета: метель/гололёд/мороз по области района). Устройство
повторяет паводки (i6: ретроспектива сезонов + SHAP-факторы «почему») и пожары
(i9: район×день, честный сплит, baseline-сравнение).

## Порядок запуска

```bash
# 1. Данные (описание: docs/ideas/i-winter-risk/winter-ml-data.md)
python3 scripts/winter_fetch_daily.py                 # фичи ERA5 район×день
python3 scripts/winter_labels_storm_archive.py        # разметка (архив Казгидромета)

# 2. Модель (venv с ml/i-winter-risk/requirements.txt)
.venv/bin/python ml/i-winter-risk/winter_ml.py train    # модель + metrics.json
.venv/bin/python ml/i-winter-risk/winter_ml.py seasons  # ретроспектива сезонов → фронтенд
.venv/bin/python ml/i-winter-risk/winter_ml.py today    # прогноз на сегодня → фронтенд
```

## Артефакты

- `data/processed/winter_lgbm.txt` — модель;
- `ml/i-winter-risk/metrics.json` — AUC/AP/capture против формульного baseline + приоры;
- `frontend/public/data/winter-ml-seasons.json` — `{сезон: {район: {score, proba, factors}}}`:
  `score` — процентиль района внутри сезона 0–100 (для карты: сырые вероятности
  малы при базовой частоте ~3% и на общей шкале выглядели бы пусто), `proba` —
  средняя вероятность опасного дня в %, показывается в карточке района;
- `frontend/public/data/winter-ml-today.json` — скор и факторы на сегодня.

## Честность

- Сплит по сезонам: train — зимы до 2024–25, test — 2025–26; приоры считаются
  только по train (без утечки).
- Target — предупреждение синоптика, а не факт инцидента (ограничение описано
  в winter-ml-data.md; факты ДТП/закрытий трасс — следующий шаг).
- Без данных ML-режим на фронтенде просто задизейблен — формульный индекс
  продолжает работать.
- Прогноз «сегодня» работает только в зимний сезон (ноябрь–март): модель обучена
  на зимних днях, летом `today` отказывает, а фронт скрывает пункт «сегодня» —
  экстраполяция вне области применимости не показывается.
