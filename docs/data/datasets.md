# Датасеты И-6 (по разделу 9 ТЗ)

Для каждого источника: откуда, структура, как обработан, какие признаки даёт, ограничения, почему подходит. Все данные открытые; ничего закрытого или персонального не используется.

## 1. Населённые пункты СКО — OSM Overpass

- **Источник:** OpenStreetMap, Overpass API (`place=city|town|village|hamlet` в границе `ISO3166-2=KZ-59`). Скрипт: `scripts/download_settlements.py`.
- **Структура:** `data/raw/settlements_kz-sev.csv` — name, lat, lon, population (тег OSM, есть не у всех), place.
- **Обработка:** дедупликация по имени при загрузке в БД (тёзки различаются координатами); 606 НП.
- **Признаки:** координаты (объект скоринга), население (входит в приоритет мер и в модель).
- **Ограничения:** население OSM неполное/устаревшее (замена — КАТО/БНС на основной программе); состав НП по OSM, не по официальному справочнику.
- **Почему подходит:** полное покрытие региона с координатами без запросов в госорганы.

## 2. Реки — OSM Overpass

- **Источник:** OSM `waterway=river` по региону. Скрипт: `scripts/compute_terrain_features.py` (зеркала Overpass с ретраями).
- **Структура:** ~7 700 точек полилиний рек (каждый 3-й узел геометрии).
- **Признаки:** `dist_to_river_km` — расстояние до ближайшей реки (поиск по сетке 0.25°).
- **Ограничения:** малые ручьи/логи не размечены в OSM — а талые воды идут и по ним; интенсивность разметки рек в OSM по СКО неравномерна.
- **Почему подходит:** главные механизмы ущерба-2024 (Есиль и притоки) покрыты.

## 3. Рельеф — Copernicus DEM GLO-90 (через Open-Meteo Elevation API)

- **Источник:** Open-Meteo Elevation API (внутри — Copernicus DEM). Батчи по 50 точек, backoff на 429.
- **Структура/признаки:** `elevation_m` (высота НП), `elev_above_river_m` (превышение над ближайшей точкой реки — ключевой физический фактор риска).
- **Ограничения:** 90 м разрешение сглаживает микрорельеф (дамбы, насыпи не видны); высота по центроиду НП, не по кварталам.
- **Почему подходит:** превышение над рекой — главный статический предиктор затопления; DEM глобально консистентен.

## 4. Метео — ERA5-Land monthly (Copernicus CDS)

- **Источник:** `reanalysis-era5-land-monthly-means`, 2000–2024, ноябрь–апрель, бокс СКО, сетка 0.1°. Скрипты: `scripts/download_era5.py`, `scripts/compute_era5_features.py`. Требуется бесплатный ключ CDS.
- **Структура:** NetCDF: t2m, tp, sd (снегозапас, м вод. экв.), smlt, sro; 150 месячных слоёв 36×66.
- **Признаки:** `swe_mar2024_mm`, `swe_mar_pct_norm` (март-2024 к норме 2000–2023; по СКО медиана **147 %**), `precip_novmar_mm`, `precip_novmar_pct_norm` — по ближайшей ячейке к НП.
- **Ограничения:** реанализ, не наблюдения станций; 0.1° ≈ 9 км — локальные аномалии сглажены; месячные средние скрывают резкие оттепели (лечится hourly-датасетом).
- **Почему подходит:** физический драйвер паводка-2024 (аномальный снегозапас) выражен количественно и виден в данных.
- **Ретроспектива:** тот же файл питает сравнение сезонов 2010–2024 (`compute_season_features.py` → `score_years.py`): модель, обученная на 2024, скорит погоду каждого года; в UI — селект сезона и график снегозапаса по годам. Продуктовый режим «текущий сезон live» (ERA5 обновляется с лагом ~5 дней) — план основной программы.

## 5. Таргет — затопления-2024 (собственная разметка)

- Полная методика, источники и стратегия негативов: [flood-labels-methodology.md](flood-labels-methodology.md).
- **Структура:** `data/raw/labels_positives.csv` (17 позитивов + ссылка-источник на каждый), `data/raw/labels_2024.csv` (name, flooded — 519 строк).
- **Ограничения (ключевое для жюри):** PU-разметка — негативы предполагаемые; медиа-смещение к крупным сёлам (признак population может частично кодировать это смещение); одно событие-2024.

## 6. Границы областей — geoBoundaries ADM1 (карта)

- **Источник:** geoBoundaries gbOpen KAZ ADM1 (ODbL, атрибуция на карте), упрощённая геометрия, `frontend/public/geo/kz-regions.geojson`.
- **Ограничения:** границы 2017 г. — 16 регионов, без Абайской/Жетысу/Улытау; на визуализацию СКО не влияет, заменить при выходе за пилотный регион (синхронно с сидером `DbSeeder`).
