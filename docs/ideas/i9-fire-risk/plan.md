# И-9. Пожарный риск-скоринг — план работ и оценка по параметрам хакатона

Роль в общем плане (см. [docs/plan/development-plan.md](../../plan/development-plan.md)): **запасной кейс** (гейт И-6 пройден — активация не потребовалась) и **второй модуль платформы** в слайде «Дальнейшие шаги».

**Статус 16.07 — реализовано и live, и ML:** ситуационная карта по 174 районам (live-метео + исторический приор FIRMS, очаги за 24 ч, ветер, слои GIBS, обновление раз в час) **плюс дневная LightGBM-модель** (район×день, 186 тыс. строк, честный тест-2025: AUC 0.832 против 0.686 у композита, capture@top-20% 55% против 36%) с режимом «ML-прогноз на сегодня» на карте — см. [ml/i9-fire-risk/](../../../ml/i9-fire-risk/). План ниже (сетка 10×10 км, NDVI) — следующий шаг детализации на основной программе.

## Цель MVP (к 17.07, если активирован)

Для 1–2 областей (Абайская + Костанайская): прогноз вероятности возгорания по ячейкам сетки 10×10 км на завтра + объяснение + приоритизация патрулей + оповещения по свежим очагам FIRMS. Живое демо в существующем приложении.

## План работ (сжатый: активация 12.07 → сдача 17.07)

### Данные (B, 12–13.07) — все источники безрисковые
| Датасет | Источник | Назначение | Риск доступа |
| --- | --- | --- | --- |
| Очаги пожаров 2001–2026 (таргет) | NASA FIRMS (MODIS C6.1 + VIIRS), открытый API/архив CSV | обучение, история | **нулевой** |
| Метео: t, ветер, влажность, осадки | ERA5 / ERA5-Land (CDS API) | признаки | низкий |
| Растительность: NDVI и динамика | MODIS MOD13 / Sentinel-2 | признак сухостоя | низкий |
| Тип растительности | ESA WorldCover | признаки | низкий |
| Дороги, ЛЭП, НП | OSM | антропогенные признаки | низкий |

### ML (B, 13–15.07)
1. Сетка 10×10 км по областям; таргет: «был ли очаг FIRMS в ячейке в день D» (сезоны апрель–октябрь).
2. Признаки: метео дня и накопленные (дни без осадков, сумма t), NDVI и его падение с пика, тип растительности, антропогенные расстояния, история очагов ячейки, день года.
3. Baseline: **канадский FWI** (стандарт пожарной метеоопасности) — считается по формуле из ERA5.
4. Модель: LightGBM с весами классов (очаги редки); прирост к FWI — главный тезис слайда AI/ML.
5. Валидация по годам: train ≤ 2023, test 2024–2025 (честная временная схема — сильный аргумент для жюри).
6. Метрика пользователя: precision@k / capture rate — «какая доля реальных очагов попала в топ-20 % приоритетных ячеек».
7. SHAP per-ячейка → top-5 факторов в БД.
8. Nowcasting: пуллинг FIRMS каждые 3 ч; фильтр устойчивых «горячих точек» (факелы, свалки, ТЭЦ) по истории повторяемости.

### Приложение (A, 13–15.07) — переиспользование ~80 %
1. Слой сетки (GeoJSON ячеек) на существующей карте; цвет = риск на завтра.
2. Деталка ячейки: скор, «Почему» (SHAP), ближайшие НП, история очагов.
3. Очередь патрулей вместо очереди мер (та же таблица/статусы: предложено → назначен патруль) — human-in-the-loop.
4. Лента свежих очагов FIRMS + оповещение «очаг ≤ N км от НП» (в UI; SMS/Telegram — в «дальнейших шагах»).
5. Демо-путь: логин → карта риска на завтра → топ-ячейки → «почему» → назначить патруль → лента очагов.

## Оценка соответствия параметрам хакатона

### Критерии ТЗ §12 (прогноз честный)

| Критерий | Макс | Прогноз | Обоснование |
| --- | --- | --- | --- |
| Понимание проблемы | 15 | **12–13** | Боль ежегодная и понятная (Абайская-2023: ~60 тыс. га, жертвы), но резонанс у жюри ниже паводков-2024 |
| Ценность решения | 15 | **12–13** | Приоритизация патрулей и авиачасов — измеримо (capture rate); экономика слабее задокументирована, чем у паводков |

| AI/ML-подход | 20 | **17–18** | Честная временная валидация (train ≤2023 / test 2024–25), сравнение с общепринятым FWI-baseline, борьба с дисбалансом классов |

### Требования и запреты ТЗ

- ✅ Одна проблема; пользователь: диспетчер лесхоза / ДЧС области
- ✅ Почему не автоматизация: показ очагов FIRMS — автоматизация; ML — **прогноз на завтра** и фильтр ложных срабатываний
- ✅ Компоненты из §6: прогнозирование риска + скоринг/ранжирование + кластеризация (исторические «пожарные коридоры») + поиск аномалий (фильтр ложных очагов)
- ✅ Explainability §7 и роль человека §8: патруль назначает диспетчер
- ✅ Реальные открытые данные без разрешений — самое чистое соответствие §5/§9 из всех идей
- ✅ Не повторяет Decentrathon 5.0; не портал; не чёрный ящик
