# И-9. Степные/лесные пожары — ценность для государства и AI/ML-подход

## Какую боль закрываем

Ежегодные степные и лесные пожары (Абайская обл. 2023: ~60 тыс. га, погибли лесники). Территория огромная, патрулей, техники и авиации мало. Сегодня решение «куда отправить патруль» принимается интуитивно, а очаги из NASA FIRMS никто системно не доводит до сельских акиматов в реальном времени.

Конкретные разрывы:
- **Нет приоритизации патрулирования**: лесхозы и ДЧС объезжают территорию по графику, а не по риску.
- **Раннее обнаружение не автоматизировано**: спутниковые очаги (FIRMS обновляется каждые ~3 часа) доходят до района с опозданием или не доходят.
- Силы и средства (авиачасы Казавиалесоохраны, техника) распределяются по областям без объективного риска.

## Кому и как продавать

| Стейкхолдер | Что получает | Питч |
| --- | --- | --- |
| **Лесхоз / Комитет лесного хозяйства (МЭПР)** | Ежедневная карта риска + маршруты патрулей | «Патрулируйте по риску, а не по графику — те же люди, меньше пропущенных возгораний» |
| **ДЧС области** | Раннее оповещение по очагам рядом с НП | «Очаг в 8 км от села — SMS акиму округа через 5 минут после пролёта спутника» |
| **Акимат района/округа** | Список НП в зоне риска на завтра, готовность | «Знайте с вечера, где завтра красная зона» |
| **Казавиалесоохрана** | Приоритизация авиапатрулирования | «Авиачас стоит дорого — летайте туда, где риск, экономьте моточасы» |

**Продающая рамка:** та же, что у И-6 — превентивное управление вместо реактивного, «очередь действий с объяснением». Вместе они продаются как **единая платформа управления природными рисками для акиматов** с модулями (паводок весной, пожар летом) — одна карта, одна модель работы, разные сезоны. Это усиливает обе идеи на защите.

**Козырь идеи:** данные полностью открыты и реальны (FIRMS API + ERA5) — никакой синтетики, модель обучается на 20+ годах фактических очагов по Казахстану.

## Где именно AI/ML

Простое отображение очагов FIRMS — автоматизация, жюри это не засчитает. ML — в **прогнозе на завтра** и фильтрации шума:

1. **Прогноз вероятности возгорания по ячейкам сетки 10×10 км на 1–3 дня.**
   - **Таргет:** очаги FIRMS (MODIS/VIIRS, архив 2001–2026) по Казахстану.
   - **Признаки:** метео ERA5 (температура, ветер, влажность, дни без осадков), состояние растительности/сухостой (NDVI из MODIS/Sentinel-2, его сезонная динамика), тип растительности (ESA WorldCover), антропогенные факторы (близость дорог, ЛЭП, НП, полей — поджоги стерни), день года, история очагов в ячейке.
   - **Модель:** градиентный бустинг с учётом редкости класса (взвешивание/undersampling); валидация по годам (train ≤2023, test 2024–2025).
2. **Канадский Fire Weather Index (FWI) как baseline и признак** — общепринятая формула пожарной метеоопасности: даёт сильный бенчмарк («наша модель лучше FWI на X %») и понятную объяснимость.
3. **Nowcasting-оповещения:** пуллинг FIRMS API каждые 3 часа → очаг ближе N км от НП/леса → оповещение с ML-фильтром ложных срабатываний (факелы месторождений, свалки, ТЭЦ — устойчивые «горячие точки» отсеиваются по истории).
4. **Кластеризация исторических очагов (DBSCAN/KDE)** → «пожарные коридоры» для планирования минерализованных полос.
5. **Explainability:** SHAP → *«Риск 84/100: 21 день без осадков, ветер 18 м/с, сухостой (NDVI упал на 40 % с мая), 7 исторических очагов в ячейке, рядом трасса»*.
6. **Human-in-the-loop:** маршруты патрулей предлагает система, утверждает лесничий/диспетчер ДЧС.

## Спутниковый мониторинг самых рискованных мест

Два независимых спутниковых контура + один для валидации:

1. **Обнаружение (уже горит) — NASA FIRMS.** MODIS + VIIRS (375 м), пролёты над КЗ каждые ~3–6 ч, открытый API. Наша добавка поверх сырых очагов: приоритизация по нашей риск-карте (очаг в топ-риск ячейке и ≤N км от НП → оповещение первым) и отсев устойчивых ложных точек (факелы месторождений, ТЭЦ, свалки) по истории повторяемости.
2. **Предрасположенность (ещё не горит, но сухо) — NDVI.** Sentinel-2 (10 м, ~5 дней, мешают облака) или MODIS MOD13 (250 м, 16 дней, стабильно): падение NDVI от майского пика = накопление сухого топлива. Еженедельное обновление «индекса сухостоя» по ячейкам → признак модели → карта «где загорится при искре» обновляется сама → маршруты патрулей.
3. **Валидация — выгоревшие площади.** dNBR по Sentinel-2 или готовый MODIS MCD64A1 (burned area): исторические шрамы пожаров подтверждают, что топ-риск ячейки действительно горели.

Ограничения: оптика слепа под облаками; FIRMS пропускает короткие возгорания между пролётами; геостационаров с полезным разрешением над КЗ нет. Архитектура приложения переиспользуется без изменений: сетка ячеек → метрики → карта → очередь патрулей.

## MVP на 8 дней

Одна–две области (Абайская + Костанайская), сетка ячеек, скоринг на завтра, метрика precision@k («какая доля реальных очагов попала в топ-20 % приоритетных ячеек»), карта и оповещения в существующем приложении. Данные качаются скриптами без регистраций и разрешений — самый низкий риск по данным из всех идей.
