# Данные для зимней ML-модели (район × день)

Цель — заменить формульный зимний индекс обучаемой моделью по образцу пожарной
(`ml/i9-fire-risk`): LightGBM предсказывает вероятность опасного зимнего дня
для района, объяснения — SHAP. Ниже — источники, структура, обработка и
ограничения (формат по требованиям ТЗ, раздел «Данные»).

## Разметка (target)

**Архив штормовых предупреждений Казгидромета** —
<https://www.kazhydromet.kz/ru/storm_archive>. Официальные предупреждения
(метель, гололёд, мороз, сильный ветер, снег) по 19 регионам за все зимы.

- Сбор: `python3 scripts/winter_labels_storm_archive.py` → `data/raw/winter_storm_warnings.csv`
  (колонки `date, region_iso, region, blizzard, ice, frost, wind, snow, text`).
- Сайт — SPA: если автопоиск API не сработал, эндпоинт подсматривается один раз
  в DevTools → Network и передаётся через `--url` (инструкция в шапке скрипта).
- Типы явлений извлекаются из текста по ключевым словам; сырой текст
  сохраняется в колонке `text` для проверки.

Резервные/дополнительные источники разметки (на основную программу):

- **ДТП с геопривязкой** — карта ДТП Комитета правовой статистики
  (<https://qamqor.gov.kz/accident>, дашборды <https://qamqor.gov.kz/stats>):
  всплеск ДТП в гололёдный день как независимый target.
- **Закрытия трасс КазАвтоЖол** — публикуются новостями (теги «закрытие трасс»
  на baq.kz, tengrinews.kz, informburo.kz): парсинг + NLP-извлечение
  «дата, участок, регион».
- **WIS2-in-a-box Казгидромета** — <https://wis2box.kazhydromet.kz/oapi> —
  стандартный OGC-API с наблюдениями станций (фактическая погода, коды явлений).

## Признаки (features)

**ERA5 через Open-Meteo Archive API** (без ключа) — дневные ряды по центроидам
174 районов за сезоны 2020–21 … 2025–26:

- Сбор: `python3 scripts/winter_fetch_daily.py` → `data/processed/winter_daily_features.csv`.
- Колонки: `tmin, tmax, wmax, gusts, snowfall, rain, precip` + прокси
  `freezing_rain` (дождь ≥ 0.5 мм при tmin < 0) и `blizzard_like`
  (снегопад ≥ 1 см при ветре ≥ 10 м/с).
- Каждому району по центроиду приписана область (`region_iso`) — по ней
  выполняется join с разметкой (предупреждения объявляются на область).
- Новые области 2022 года (Абай, Жетісу, Ұлытау) сведены к родительским
  (ВКО, Алматинская, Карагандинская) — границы ADM1 в проекте старого деления.

## Сборка датасета и обучение

Строка = район × день. `target = 1`, если по области района в этот день было
предупреждение (отдельные модели или мультилейбл по `blizzard/ice/frost`).
Честный сплит по сезонам: train — зимы до 2024–25, test — 2025–26 (как в
пожарной модели). Дальше — копия пайплайна `ml/i9-fire-risk/fire_ml.py`
(train/today, metrics.json, экспорт скора в `frontend/public/data/`).

## Ограничения

- Предупреждение — прогноз синоптика, а не факт инцидента: модель учится
  предсказывать «день, который сочли опасным». Для фактов нужны ДТП/закрытия
  трасс (см. резервные источники).
- Разметка на уровне области огрубляет district-level сигнал: все районы
  области в день предупреждения получают target = 1.
- Центроид района может попасть в соседнюю область на извилистых границах
  (скрипт печатает число непривязанных районов; фактически — единицы).
